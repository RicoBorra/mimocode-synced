# Plan: Adapt opencode-synced for MiMo Code

## Goal

Adapt the forked `opencode-synced` plugin to work with MiMo Code, changing paths, names, and behavior to match MiMo Code's conventions while preserving all core sync functionality.

## Key Findings

- **Plugin API**: MiMo Code uses the same `@opencode-ai/plugin` package — no API changes needed
- **Path mapping**: Centralized in `src/sync/paths.ts` — minimal code changes
- **Session DB**: `mimocode.db` should NOT be synced (dream/distill dependency) — already excluded by default via `includeSessions: false`
- **Scope**: Core sync first (config + skills + .agents), full feature parity minus sessions

## Security & Safety

1. **SQLite DB**: Not synced by default (`includeSessions: false`). Dream/distill dependencies protected.
2. **Lock mechanism**: Existing `withSyncLock()` preserved — prevents concurrent writes.
3. **Secrets**: 1Password backend support preserved unchanged.
4. **Env var flexibility**: `MIMOCODE_SYNC_APP_NAME` allows future dual-support.
